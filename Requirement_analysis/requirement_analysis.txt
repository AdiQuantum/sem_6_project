1. Introduction

1.1 Purpose

The purpose of this document is to describe the functional and non-functional requirements for the development of an **E-Commerce Website using the MERN Stack (MongoDB, Express.js, React.js, and Node.js)**.

The platform will allow users to browse products, register accounts, add products to a cart, place orders, and make payments online.

This document will serve as a guideline for developers, designers, and stakeholders involved in the development and maintenance of the system.


1.2 Scope

The proposed system is a **web-based e-commerce platform** that enables users to purchase products online.

The website will include:

User registration and authentication
Product browsing and searching
Shopping cart functionality
Online order placement
Payment gateway integration
Admin panel for managing products and orders

The system will be developed using the **MERN stack**, ensuring scalability, high performance, and efficient data management.

3. System Features



3.1 User Registration

Description

Allows new users to create an account on the platform.

Functional Requirements

Users must enter:

  * Name
  * Email
  * Password
  * Phone number

System must:

  * Validate input fields
  * Encrypt passwords
  * Store user data in MongoDB


3.2 User Login

Description

Allows registered users to access their accounts.

Functional Requirements

* User enters email and password
* System validates credentials
* System generates JWT token
* User session is created


3.3 Product Catalog

Description

Displays available products.

Functional Requirements

* View all products
* View product details
* Filter products by category
* Search products by name

Product details include:

* Product name
* Price
* Description
* Rating
* Image
* Stock availability


3.4 Shopping Cart

Description

Allows users to manage products before purchasing.

Functional Requirements

* Add products to cart
* Remove products from cart
* Update product quantity
* View cart summary
* Calculate total price

3.5 Order Management

Description

Handles order placement and tracking.

Functional Requirements

Users can:

* Place orders
* View order history
* Track order status

System must:

* Generate unique order ID
* Store order details in database
* Update order status


3.6 Payment Processing

Description

Allows users to make secure online payments.

Functional Requirements

Supported payment methods:

* Credit/Debit Cards
* UPI
* Net Banking

System must:

* Verify payment
* Confirm order after successful payment
* Store payment information securely


3.7 Admin Panel

Description

Provides control panel for administrators.

Functional Requirements

Admin can:

* Add new products
* Update product information
* Delete products
* Manage orders
* View user accounts

5. Non-Functional Requirements



5.1 Performance Requirements

* Page load time must be less than **3 seconds**
* System must support **multiple concurrent users**
* API response time must be under **500 ms**


5.2 Security Requirements

* Passwords must be **encrypted using bcrypt**
* Authentication via **JWT tokens**
* Use **HTTPS** for secure communication
* Protect against:

  * SQL/NoSQL Injection
  * XSS attacks
  * CSRF attacks


5.3 Usability Requirements

* User-friendly interface
* Responsive design for mobile devices
* Easy navigation

5.4 Reliability Requirements

* System uptime should be **99%**
* Database backup must be maintained

5.5 Maintainability Requirements

* Modular code structure
* Well-documented APIs
* Use version control (Git)

 System Architecture

The system follows **three-tier architecture**.

 1. Presentation Layer

 React.js frontend

 2. Application Layer

* Node.js
* Express.js APIs

 3. Data Layer

* MongoDB database

Data Flow:

User → React Frontend → Express API → MongoDB → Response to User
